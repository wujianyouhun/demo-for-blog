# 11 - GeoAI SAM Web 平台使用手册

本文面向实际标注人员，说明 Web 平台从启动到导出标签的完整操作。每个步骤包含操作、预期结果、后台处理和常见失败原因。

## 1. 启动前准备

### 1.1 确认默认影像

平台默认读取项目 `data/` 目录下的 GeoTIFF。把 `.tif` 或 `.tiff` 放到：

```text
data/
```

启动后前端会从 `/api/config` 获取 `data_dir` 和 `available_images`。若 `data/` 中有影像，会自动填入第一个文件；也可以在“文件路径”里手动输入绝对路径。

### 1.2 检查模型和依赖

推荐先使用 `vit_l`。如果显存不足，用 `vit_b`；如果显存充足且追求更细边界，用 `vit_h`。

点标注和框标注至少需要：

| 依赖 | 作用 |
| --- | --- |
| `torch` | 模型推理 |
| `samgeo` | 遥感 SAM 封装 |
| `rasterio` | GeoTIFF 读取和坐标转换 |
| `fastapi` / `uvicorn` | 后端服务 |
| `titiler` | 影像瓦片服务 |

文本标注还需要 GroundingDINO、LangSAM 或 `geoai-py` 相关能力。若依赖不可用，接口会返回 503，并显示具体缺失项。

### 1.3 检查模型权重

权重目录：

```text
models/
```

常见 SAM1 文件：

| 模型 | 文件名 | 预期大小 |
| --- | --- | --- |
| `vit_b` | `sam_vit_b_01ec64.pth` | 约 376MB |
| `vit_l` | `sam_vit_l_0b3195.pth` | 约 1192MB |
| `vit_h` | `sam_vit_h_4b8939.pth` | 约 2448MB |

如果文件明显偏小，说明下载不完整。删除后重新运行：

```powershell
python download_models.py --vit_l
```

## 2. 启动服务

在项目根目录执行：

```bash
python start_web.py
```

启动成功后访问：

```text
http://127.0.0.1:5173
```

后端 API 文档：

```text
http://127.0.0.1:8011/docs
```

启动后控制台应显示前端和后端地址。默认前端为 `http://127.0.0.1:5173`，后端为 `http://127.0.0.1:8011`。若端口冲突，修改 `start_web.py` 中的 `BACKEND_PORT` 或 `FRONTEND_PORT`，并同步修改 `frontend/vite.config.js` 的代理端口。

## 3. 页面区域说明

| 区域 | 说明 |
| --- | --- |
| 顶部栏 | 显示平台名称和当前会话状态 |
| 左侧面板 | 影像加载、标注模式、后处理、矢量导出 |
| 地图主区 | 显示 GeoTIFF 瓦片、点、框和 Mask |
| 底部状态栏 | 显示经纬度、影像尺寸、CRS、Mask 状态 |
| Toast 提示 | 显示加载、分割、导出和错误消息 |

刚打开页面时，左侧只显示影像路径、模型、SAM 版本和“加载影像”按钮。

## 4. 加载影像

### 4.1 操作步骤

1. 在“文件路径”中确认 GeoTIFF 路径。
2. 选择模型：默认 `vit_l`。
3. 选择 SAM 版本：默认 `SAM 1`。
4. 点击“加载影像”。
5. 等待地图自动缩放到影像范围。

### 4.2 正确结果

加载成功后：

| 界面位置 | 正确表现 |
| --- | --- |
| 左侧影像信息 | 显示文件名、宽高、CRS、bounds |
| 地图主区 | 影像瓦片可见，视图自动定位 |
| 顶部状态 | 显示会话 ID |
| 左侧工具 | 出现标注模式、后处理、矢量导出 |

后台调用：

```http
POST /api/image/load
```

后端返回 `tile_url`、`tile_json_url`、`mask_extent`、`session_id` 等信息。`mask_extent` 是 EPSG:3857 范围，用于前端定位影像和叠加 Mask。

### 4.3 常见失败

| 现象 | 原因 | 处理 |
| --- | --- | --- |
| 文件不存在 | 路径错误或盘符不可访问 | 检查路径 |
| 影像加载失败 | `rasterio` 不可用或文件损坏 | 检查依赖和 GeoTIFF |
| 地图不定位 | 影像缺少 CRS 或 bounds | 换用带地理参考的 GeoTIFF |

## 5. 点标注

点标注适合快速指定目标区域，尤其是目标边界不确定、需要多次交互修正时。

### 5.1 操作步骤

1. 点击“点标注”。
2. 在目标内部单击，添加前景点。
3. 按住 `Shift` 并单击，添加背景点。
4. 双击地图，或点击“执行点分割 (Enter)”。
5. 等待绿色半透明 Mask 叠加。

### 5.2 正确结果

| 结果 | 说明 |
| --- | --- |
| 地图出现绿色半透明区域 | 后端返回了分割 Mask |
| 点标记被清除 | 本次提示已经完成推理 |
| 底部显示 “Mask 已生成” | 可以继续后处理或导出 |

后台调用：

```http
POST /api/predict/point
```

请求参数：

```json
{
  "session_id": "会话 ID",
  "points": [[lon, lat]],
  "labels": [1]
}
```

`labels=1` 表示前景点，`labels=0` 表示背景点。

### 5.3 结果判断

正确的点标注结果应覆盖目标主体，不应大量覆盖相邻背景。如果结果偏大，增加背景点；如果结果漏分，增加前景点；如果目标形状明确，建议改用框标注。

## 6. 框标注

框标注是建筑物、水体、光伏板等边界明确目标的推荐方式。

### 6.1 操作步骤

1. 点击“框标注”。
2. 在地图上按住鼠标左键拖出矩形。
3. 松开鼠标后自动触发分割。
4. 等待 Mask 叠加。

### 6.2 正确结果

| 结果 | 说明 |
| --- | --- |
| 拖动时显示蓝色虚线框 | 前端正在绘制提示框 |
| 松开后出现绿色 Mask | 后端完成框提示分割 |
| 框被清除 | 本次分割已经结束 |

后台调用：

```http
POST /api/predict/box
```

请求参数：

```json
{
  "session_id": "会话 ID",
  "box": [lon1, lat1, lon2, lat2]
}
```

后端会把地理坐标转换成像素坐标，并统一排序为 `[xmin, ymin, xmax, ymax]`，避免地图坐标方向与像素行列方向不一致。

## 7. 文本标注

文本标注使用 Grounded-SAM 相关能力，适合批量预标注和开放词汇目标。

### 7.1 操作步骤

1. 点击“文本标注”。
2. 输入英文提示词，例如：

```text
building
water. lake. river
road
tree. vegetation
```

3. 在地图上点击一次。
4. 等待检测和分割结果。

### 7.2 正确结果

| 返回 | 说明 |
| --- | --- |
| 绿色 Mask | 文本模型检测到目标并驱动 SAM 分割 |
| 400 | 未生成 Mask，通常是提示词不匹配或阈值过高 |
| 503 | 文本标注依赖或基础 SAM 模型不可用 |

后台调用：

```http
POST /api/predict/text
```

请求参数：

```json
{
  "session_id": "会话 ID",
  "text": "building",
  "box_threshold": 0.25,
  "text_threshold": 0.25
}
```

## 8. 后处理

后处理用于把原始 Mask 调整成更适合训练标签的形态。

### 8.1 操作步骤

1. 先完成一次点、框或文本分割。
2. 设置后处理参数。
3. 点击“执行后处理”。
4. 等待新的 Mask 叠加。

### 8.2 参数说明

| 参数 | 默认值 | 作用 |
| --- | --- | --- |
| 最小面积 | 200 | 移除小碎斑 |
| 填充孔洞 | true | 填充目标内部孔洞 |
| 平滑 Sigma | 1.5 | 平滑边界 |
| 开运算半径 | 2 | 去除毛刺和小噪声 |
| 闭运算半径 | 3 | 连接小断裂 |

![Mask 后处理对比](images/mask-postprocessing-comparison.png)

如果后处理后目标变小，降低“最小面积”或“开运算半径”。如果孔洞仍明显，保持“填充孔洞”开启并适当增大闭运算半径。

## 9. 矢量导出

矢量导出把当前 Mask 转成 GIS Polygon。

### 9.1 操作步骤

1. 确认已经生成 Mask。
2. 设置“最小面积”。
3. 选择导出格式。
4. 点击“导出矢量”。
5. 点击“下载文件”。

### 9.2 格式说明

| 格式 | 说明 |
| --- | --- |
| GeoJSON | 适合 Web、调试和版本管理 |
| GeoPackage | 适合生产保存，单文件 |
| Shapefile | 传统 GIS 软件兼容性好 |

![矢量化结果](images/vectorization-result.png)

正确导出后，左侧会显示多边形数量和输出文件路径。

## 10. 整幅影像处理

当影像范围很大时，不应把整幅 GeoTIFF 一次性送入 SAM。Web 平台提供“整幅影像处理”面板，后端会把原图切成带重叠的瓦片，逐瓦片推理、矢量化，再合并成一个结果文件。

### 10.1 操作步骤

1. 先加载影像。
2. 在“整幅影像处理”中选择模式。
3. 设置瓦片尺寸、重叠像素、最小面积和导出格式。
4. 点击“启动整图处理”。
5. 等待进度达到 100%。页面会显示当前阶段、百分比和瓦片数量。
6. 点击“下载整图结果”。

后台接口：

```http
POST /api/process/full
GET /api/process/status?task_id=...
GET /api/process/download?task_id=...
```

### 10.2 模式说明

| 模式 | 处理范围 | 适用场景 |
| --- | --- | --- |
| 文本 | 遍历整幅影像全部瓦片 | 建筑、水体、道路等同类目标预提取 |
| 自动 | 遍历整幅影像全部瓦片 | 没有明确提示词时的候选目标发现 |
| 点提示 | 只处理包含提示点的瓦片 | 局部目标精修，不适合用一个点代表整幅图 |
| 框提示 | 只处理与提示框相交的瓦片 | 大范围框选区域内提取 |

### 10.3 参数建议

| 参数 | 推荐值 | 说明 |
| --- | --- | --- |
| 瓦片尺寸 | 2048 | 适合 6GB 显存的默认值 |
| 重叠像素 | 256 | 减少跨瓦片目标被切断 |
| 最小面积 | 50~200 | 过滤碎斑，建筑可适当增大 |
| 格式 | GeoPackage | 单文件，适合大结果 |

如果显存不足，把瓦片尺寸降到 `1024`。如果瓦片边界漏检明显，增大重叠到 `384` 或 `512`。

文本模式和自动模式首次运行会先加载模型，可能在第一个瓦片前等待较久。此时任务消息会显示“正在加载文本标注模型”或“正在加载 SAM 自动分割模型”；模型加载完成后才会进入逐瓦片处理。

### 10.4 输出位置

每个整图任务输出到：

```text
output/full_image/<task_id>/
```

其中：

| 内容 | 说明 |
| --- | --- |
| `tiles/` | 临时瓦片 GeoTIFF，保留原图地理参考 |
| `full_image_result.gpkg` | 默认整图矢量结果 |
| `full_image_result.geojson` | 选择 GeoJSON 时生成 |
| `full_image_result.shp` | 选择 Shapefile 时生成，下载接口会打包为 zip |

## 11. 推荐生产流程

高质量标注流程：

```text
加载影像
  -> 框标注获得初始 Mask
  -> 点标注补充修正
  -> 后处理
  -> 导出 GeoPackage
  -> QGIS/ArcGIS 抽查和修订
```

快速预标注流程：

```text
加载影像
  -> 文本标注批量查找目标
  -> 人工检查
  -> 后处理
  -> 导出 GeoJSON
```

如果文本标注依赖暂不可用，使用框标注作为主流程。

## 12. 常见错误

| 错误 | 原因 | 处理 |
| --- | --- | --- |
| 404 文件不存在 | 影像路径错误 | 检查盘符和中文目录 |
| 503 缺少基础 SAM 依赖 | `torch` 或 `samgeo` 不可用 | 安装依赖并重启服务 |
| 503 模型文件不完整 | `.pth` 权重下载不完整 | 删除后重新下载 |
| 400 点坐标数量与标签数量不一致 | 前端或接口参数错误 | 清除点后重新标注 |
| 400 坐标转换失败 | CRS 或地图范围异常 | 检查影像 CRS |
| 后处理失败 | 当前 Mask 为空或形状异常 | 重新执行分割 |
| 导出无文件 | 当前没有 Mask | 先执行分割或后处理 |
| 整图任务长时间运行 | 原图太大或模型较重 | 使用 `vit_b`、降低瓦片尺寸或优先使用文本模式 |
| 整图点提示结果很少 | 点模式只处理包含提示点的瓦片 | 改用文本模式或框选更大范围 |
