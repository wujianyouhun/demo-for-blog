# 点、框、文本：SAM 遥感标注的三种提示方式该怎么选

## 导语

SAM 的核心能力是“提示式分割”。同一张遥感影像，可以用点、框、文本三种方式驱动模型。

但在真实标注任务中，三种方式的效率和稳定性差异很大。选错模式，可能比人工勾画还折腾；选对模式，则能显著提高标签生产效率。

本文结合遥感标注项目实践，聊聊点提示、框提示、文本提示分别适合什么场景。

## 点提示：最轻量，也最依赖交互

点提示是最简单的方式：

```json
{
  "points": [[lon, lat]],
  "labels": [1]
}
```

`labels=1` 表示前景点，`labels=0` 表示背景点。

### 适合场景

点提示适合：

- 目标边界不确定。
- 需要快速探索模型反应。
- 局部目标修正。
- 前景和背景纹理相对清晰。

比如水体边缘、树冠、裸地、阴影附近的目标，可以先给一个前景点，看模型覆盖范围，再补背景点修正。

### 使用技巧

前景点不要点在边界上，最好点在目标内部稳定区域。背景点也不要离目标太远，应放在模型容易误分的相邻区域。

典型流程：

```text
一个前景点
  -> 观察 Mask
  -> 漏分则补前景点
  -> 误分则补背景点
  -> 后处理和导出
```

点提示的优点是交互轻，缺点是结果不够稳定。对于形状规则的建筑物，不如框提示直接。

## 框提示：遥感标注里的主力模式

框提示输入的是目标大致范围：

```json
{
  "box": [lon1, lat1, lon2, lat2]
}
```

后端会把经纬度转成像素坐标，再裁剪局部影像送入 SAM。

### 适合场景

框提示适合：

- 建筑物。
- 光伏板。
- 规则水池。
- 独立地块。
- 车辆、船只等边界明确目标。

它给模型提供了更强的空间约束，能显著降低误分。

### 框要怎么画

框太小，可能切掉目标边界；框太大，背景干扰会变多。

经验上，框应该略大于目标，留一点边缘上下文：

```text
目标外扩 5%~15%
```

如果目标很长，例如道路、水渠，单个框可能不够稳定，可以分段框选或改用文本/整图流程。

## 文本提示：适合批量预标注

文本提示的输入是类别描述：

```json
{
  "text": "building",
  "box_threshold": 0.25,
  "text_threshold": 0.25
}
```

系统会先用 GroundingDINO 或 LangSAM 找候选区域，再用 SAM 分割。

### 适合场景

文本提示适合：

- 同一类目标数量很多。
- 先做预标注，再人工筛选。
- 建筑物、水体、道路、光伏板等具有明确语义的对象。

推荐提示词：

| 类别 | 提示词 |
| --- | --- |
| 建筑物 | `building`, `house`, `roof` |
| 水体 | `water`, `lake`, `river` |
| 道路 | `road`, `highway`, `street` |
| 光伏板 | `solar panel`, `photovoltaic` |
| 车辆 | `car`, `truck`, `vehicle` |

### 阈值怎么调

文本模式常见两个阈值：

| 参数 | 作用 |
| --- | --- |
| `box_threshold` | 检测框置信度阈值 |
| `text_threshold` | 文本匹配阈值 |

如果漏检很多，可以适当降低阈值；如果误检很多，可以提高阈值。

建议从 `0.25` 起步，再根据结果调整。

## 三种方式怎么选

可以用下面这张表快速判断：

| 目标状态 | 推荐方式 | 原因 |
| --- | --- | --- |
| 单个目标，边界清晰 | 框提示 | 空间约束强，结果稳定 |
| 单个目标，边界模糊 | 点提示 | 便于用正负点修正 |
| 大量同类目标 | 文本提示 | 可批量生成候选 |
| 需要生产级精修 | 框提示 + 后处理 | 兼顾效率和边界质量 |
| 不知道有哪些目标 | 自动分割 | 用于探索候选区域 |

## 大影像中的提示策略

对于超大 GeoTIFF，点和框提示通常只处理局部区域。

如果想处理完整影像：

- 文本模式：遍历所有瓦片，适合提取同类目标。
- 自动模式：遍历所有瓦片，适合发现候选目标。
- 点模式：只处理包含提示点的瓦片。
- 框模式：只处理与提示框相交的瓦片。

这意味着：不要指望一个点代表整幅图。想做整图建筑物提取，应该使用文本模式或自动模式。

## 小结

点、框、文本不是谁替代谁，而是对应不同生产环节：

- 点提示用于快速探索和局部修正。
- 框提示用于高质量单目标标注。
- 文本提示用于批量预标注。

真正高效的遥感标注流程，往往是三者组合：文本先批量提取，框提示修正关键目标，点提示处理边界细节。
