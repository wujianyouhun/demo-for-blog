# 从 Mask 到 GIS 标签：遥感分割结果的后处理与矢量化

## 导语

SAM 输出的是 Mask，但 GeoAI 训练和 GIS 交付通常需要 Polygon。

如果直接把原始 Mask 转成矢量，结果往往会有很多问题：碎斑、孔洞、锯齿边界、小面积误检、无效几何。

因此，从模型结果到可用标签，中间必须有后处理和矢量化。

本文讲清楚这条链路。

## 原始 Mask 常见问题

遥感影像纹理复杂，SAM 输出的 Mask 可能出现：

- 小碎斑：阴影、噪声或纹理被误分。
- 内部孔洞：目标内部纹理不一致。
- 锯齿边界：栅格分辨率导致边缘不平滑。
- 粘连：相邻目标被连成一片。
- 面积异常：过小结果进入导出。

这些问题如果不处理，会直接影响后续训练质量。

## 后处理流程

一个常用流程是：

```text
原始 Mask
  -> remove_small_objects
  -> fill_holes
  -> gaussian smooth
  -> opening / closing
  -> area filter
  -> vectorize
```

项目中通过 `MaskPostProcessor` 封装这些操作。

典型参数：

| 参数 | 含义 |
| --- | --- |
| `min_size` | 去除小于该像素数的斑块 |
| `fill_holes` | 是否填充孔洞 |
| `smooth_sigma` | 高斯平滑强度 |
| `opening_radius` | 开运算半径，去除细小噪声 |
| `closing_radius` | 闭运算半径，连接小断裂 |

参数不是越大越好。过强的平滑会改变真实边界，过高的面积阈值会删除小目标。

## 不同目标的参数建议

| 目标 | 参数倾向 |
| --- | --- |
| 建筑物 | 中等平滑，较高面积过滤 |
| 水体 | 允许较大连通区域，注意填洞 |
| 道路 | 不宜过强开运算，避免细长目标被删除 |
| 车辆 | 面积阈值要低，避免小目标丢失 |
| 光伏板 | 适当平滑，保留规则边界 |

后处理的目标不是让结果“看起来更圆滑”，而是让标签更符合业务对象。

## Mask 如何转 Polygon

矢量化的关键是把栅格区域转成几何边界。

基本流程：

```text
二值 Mask
  -> 连通区域提取
  -> 轮廓追踪
  -> 生成 Polygon
  -> 绑定 CRS
  -> 面积过滤
  -> 写出文件
```

项目里的 `MaskVectorizer` 会读取参考影像的 transform 和 CRS，确保输出几何仍然在原始坐标系下。

这一步非常关键。否则得到的只是像素坐标 Polygon，无法在 GIS 软件中和原始影像对齐。

## 导出格式选择

常见导出格式有三种：

| 格式 | 优点 | 注意 |
| --- | --- | --- |
| GeoJSON | 易读，适合 Web 调试 | 大数据量文件偏大 |
| Shapefile | 兼容传统 GIS | 多文件组成，字段名有限制 |
| GeoPackage | 单文件，支持大数据 | 推荐用于整图结果 |

如果是生产流程，建议优先使用 GeoPackage。  
如果是调试和展示，可以使用 GeoJSON。  
如果要交给传统 GIS 软件或旧流程，可以导出 Shapefile。

## 给矢量结果加属性

一个可用标签不只有几何，还应该包含属性。

例如：

| 字段 | 说明 |
| --- | --- |
| `id` | 唯一编号 |
| `area_pixels` | 像素面积 |
| `tile_id` | 来源瓦片 |
| `mode` | 标注模式 |
| `class_name` | 类别名称 |

整图瓦片化处理时，`tile_id` 很有用。它能帮助排查某个瓦片结果异常，也方便后续按区域回溯。

## 如何判断结果质量

如果有人工真值，可以用指标评估：

- IoU。
- Dice。
- Precision / Recall。
- Hausdorff Distance。

如果没有真值，也可以做人工检查：

- 是否覆盖目标主体。
- 是否明显外扩到背景。
- 小碎斑是否过多。
- 边界是否过度平滑。
- CRS 是否和原图对齐。

对于遥感标签，几何正确和坐标正确同样重要。

## 常见问题

### 1. 导出的矢量偏移

优先检查参考影像 transform 和 CRS。  
如果 Mask 是裁剪子图生成的，还要检查 `crop_offset` 是否正确放回全图位置。

### 2. Polygon 太碎

提高 `min_area`，增加开运算，或在矢量化后做面积过滤。

### 3. 目标内部有孔

启用 `fill_holes`，但要注意水体、道路等对象是否本身存在真实空洞。

### 4. Shapefile 下载后打不开

Shapefile 不是单个文件，必须同时包含 `.shp`、`.shx`、`.dbf`、`.prj` 等组件。Web 下载时建议打包成 zip。

## 小结

模型输出只是标签生产的中间结果。真正能进入训练或 GIS 交付的数据，需要经过：

```text
Mask -> 后处理 -> 矢量化 -> 属性补充 -> 格式导出 -> 质量检查
```

这条链路做好了，SAM 才能从“分割工具”变成“标签生产工具”。
