# 用 SAM 重构遥感影像标注流程：从人工勾画到半自动标签生产

## 导语

遥感影像标注一直是 GeoAI 项目里最耗时间的一环。

无论做建筑物提取、水体识别、道路检测，最终都绕不开高质量标签。传统方式通常是在 GIS 软件里逐个目标勾画 Polygon。影像范围稍大一点，标注成本就会迅速上升；不同标注员之间的边界风格也不一致，后期质检压力很大。

SAM 这类基础分割模型的出现，让遥感标注流程有了新的组织方式：人不再从零开始勾边，而是给模型一个点、一个框，甚至一句文本提示，让模型先生成 Mask，再由人工校正。

本文结合一个 GeoAI SAM 标注项目，梳理如何把 SAM 接入遥感标签生产流程。

## 传统遥感标注为什么慢

遥感标注的难点不只是“画多边形”。

一张高分辨率影像里，目标数量多、尺度变化大、边界细碎，还会遇到阴影、遮挡、纹理相似等问题。人工标注时通常要经历：

1. 打开 GeoTIFF。
2. 缩放定位目标。
3. 沿边界逐点勾画 Polygon。
4. 检查拓扑和属性。
5. 导出 Shapefile、GeoJSON 或 GeoPackage。

如果目标是建筑物、光伏板、水体边界这类高频对象，重复劳动非常明显。一个熟练标注员处理一片城区影像，可能花几个小时甚至更久。

SAM 的价值就在这里：它把“从零勾画”变成“提示 + 修正”。

## SAM 在标注流程里的角色

SAM 的输入不是遥感语义类别，而是提示：

- 一个前景点。
- 一个背景点。
- 一个边界框。
- 一个已有 Mask。

模型输出对应区域的分割 Mask。对于遥感场景，这意味着我们可以把交互动作压缩到很少：

- 建筑物：框住屋顶，自动出边界。
- 水体：在水面点一下，再用背景点排除阴影。
- 地块：用框限制范围，再让模型找边缘。

如果再接入 GroundingDINO 或 LangSAM，流程还能进一步升级为：

```text
输入文本 building
  -> 检测可能的 building 区域
  -> 对每个候选区域执行 SAM 分割
  -> 输出建筑物 Mask
```

这就是“文本描述 -> 自动检测 -> 精确分割”的遥感预标注路径。

## 一个完整的工程流程

实际项目不能只停留在模型推理。遥感标签要能进入 GIS 和训练流程，所以完整链路至少包括：

```text
GeoTIFF 影像
  -> 读取元数据和 CRS
  -> 地图瓦片显示
  -> 点/框/文本提示
  -> SAM 推理生成 Mask
  -> Mask 后处理
  -> 栅格 Mask 转矢量 Polygon
  -> 导出 GeoJSON / Shapefile / GeoPackage
```

在这个项目中，核心模块分成几层：

| 层级 | 作用 |
| --- | --- |
| `SAMWrapper` | 封装 SAM、SAM2、SAM3 的调用差异 |
| `GroundedSAMWrapper` | 文本提示标注，连接 GroundingDINO、LangSAM 等能力 |
| `MaskPostProcessor` | 去噪、填洞、平滑、面积过滤 |
| `MaskVectorizer` | 将 Mask 转为 GIS 矢量 |
| `QualityMetrics` | IoU、Dice、Hausdorff 等质量评估 |

这个分层的好处是：模型、后处理、导出可以单独替换，不会把整个系统绑死在某个脚本里。

## 三种典型使用方式

### 1. 点提示：适合局部修正

点提示的交互成本最低。用户在目标内部点一下作为前景点，如果结果覆盖到了背景，再补一个背景点。

它适合边界模糊、需要反复修正的场景，例如小水塘、树冠、阴影附近的目标。

### 2. 框提示：适合边界明确目标

框提示通常是遥感标注里最稳定的方式。建筑物、光伏板、独立地块等目标，只要用框限制住大概范围，SAM 就能生成比较好的边界。

如果追求生产效率，框标注往往是第一选择。

### 3. 文本提示：适合批量预标注

文本提示适合先粗提取一类目标，例如：

```text
building
water
road
solar panel
vehicle
```

它的优点是覆盖范围大，适合做预标注；缺点是依赖检测模型和提示词质量，结果通常还需要人工筛选。

## 为什么还需要后处理

模型输出的是 Mask，不一定直接等于可用标签。

常见问题包括：

- 小碎斑太多。
- 目标内部有孔洞。
- 边界锯齿明显。
- 相邻目标粘连。
- 面积过小的误检进入结果。

所以工程里需要把 Mask 继续处理：

```text
原始 Mask
  -> 去除小斑块
  -> 填充孔洞
  -> 平滑边界
  -> 开闭运算
  -> 面积过滤
  -> 矢量化
```

这样得到的 Polygon 才更接近训练标签或 GIS 成果数据。

## 效率提升来自哪里

SAM 辅助标注不是完全替代人工，而是改变人工介入的位置：

| 流程 | 人工主要工作 |
| --- | --- |
| 传统标注 | 从零勾画每个 Polygon |
| SAM 辅助标注 | 给提示、检查结果、少量修正 |
| 文本预标注 | 批量生成候选，人工筛选和补充 |

在目标边界清晰的场景里，人工工作量通常能减少 70% 以上。更关键的是，标签风格更一致，后续质检更容易。

## 小结

把 SAM 用到遥感标注里，关键不是“跑通一个模型”，而是把模型嵌入完整标签生产链路：

- GeoTIFF 能正常显示。
- 坐标转换不能错。
- 大影像不能一次性塞进显存。
- Mask 要能后处理。
- 最终结果要能导出为 GIS 矢量。

只有这些环节都打通，SAM 才能从一个 demo 变成真正可用的 GeoAI 标注工具。
