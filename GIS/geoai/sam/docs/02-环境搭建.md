## 02 - 环境搭建

### 前置条件

确保以下软件已安装:

| 软件 | 版本要求 | 用途 |
|------|----------|------|
| Anaconda / Miniconda | 最新版 | Python 环境管理 |
| NVIDIA 驱动 | >= 525.xx (Linux) / >= 530.xx (Windows) | GPU 加速 |
| CUDA Toolkit | 11.8 或 12.1 | PyTorch GPU 后端 |
| Git | >= 2.30 | 版本控制 (可选) |

### 第一步: 确认 Conda 环境

本项目使用的 conda 环境名为 `geoai`。确认该环境已存在并已安装 PyTorch:

```bash
conda activate geoai
python --version          # 期望 3.9 ~ 3.11
python -c "import torch; print(torch.__version__); print('CUDA:', torch.cuda.is_available())"
```

如果 `geoai` 环境不存在，需要先创建:

```bash
conda create -n geoai python=3.10 -y
conda activate geoai

# 安装 PyTorch (根据 CUDA 版本选择)
# CUDA 11.8:
pip install torch torchvision --index-url https://download.pytorch.org/whl/cu118
# CUDA 12.1:
pip install torch torchvision --index-url https://download.pytorch.org/whl/cu121
# 仅 CPU:
pip install torch torchvision
```

### 第二步: 安装项目依赖

进入项目目录后执行:

```bash
cd sam/
pip install -r requirements.txt
```

依赖分组说明:

| 分组 | 关键包 | 说明 |
|------|--------|------|
| 深度学习 | torch, torchvision | 核心计算引擎 |
| SAM | segment-anything, samgeo | SAM 模型及遥感封装 |
| 检测/语义 | groundingdino-py, transformers, openai-clip | GroundingDINO + CLIP |
| 遥感 | rasterio, geopandas, shapely, pyproj | 地理空间数据处理 |
| 图像 | numpy, Pillow, opencv-python, scikit-image, scipy | 图像处理与形态学 |
| 可视化 | matplotlib, seaborn | 结果展示 |

### 第三步: 配置环境变量（可选）

```bash
# Linux / macOS
cp .env.example .env

# Windows (PowerShell)
copy .env.example .env
```

编辑 `.env` 文件，关键配置项:

```ini
# SAM 模型版本和类型
SAM_VERSION=sam1           # sam1 / sam2 / sam3
SAM_MODEL_TYPE=vit_h       # vit_b / vit_l / vit_h

# 如果有本地模型权重，填写路径 (留空则自动下载)
SAM_CHECKPOINT_PATH=

# GPU 设备 (留空自动检测)
DEVICE=

# 检测阈值
BOX_THRESHOLD=0.25
TEXT_THRESHOLD=0.25
```

### 第四步: 验证安装

```bash
# 1. 检查配置（含模型缓存路径）
python config.py

# 2. 检查核心包导入
python -c "from geoai_sam import SAMWrapper, GroundedSAMWrapper, MaskPostProcessor, MaskVectorizer, QualityMetrics; print('全部导入成功')"

# 3. 检查 GPU 信息
python -c "
import torch
if torch.cuda.is_available():
    print(f'GPU: {torch.cuda.get_device_name(0)}')
    print(f'显存: {torch.cuda.get_device_properties(0).total_mem / 1024**3:.1f} GB')
    print(f'CUDA: {torch.version.cuda}')
else:
    print('未检测到 GPU，将使用 CPU 运行 (速度较慢)')
"

# 4. 确认模型缓存不会下载到系统 .cache
python -c "
import os, config
print('TORCH_HOME:', os.environ.get('TORCH_HOME'))
print('HF_HOME:', os.environ.get('HF_HOME'))
print('均在项目 models/ 目录下:', 'models' in os.environ.get('TORCH_HOME', ''))
"
```

### 常见问题

**rasterio 安装失败**

`rasterio` 依赖 GDAL 底层库，在 Windows 上直接 `pip install` 可能失败。推荐:

```bash
conda install -c conda-forge rasterio geopandas
```

**samgeo 找不到**

```bash
pip install samgeo
# 或
pip install geoai-py
```

**CUDA out of memory**

显存不足时，在 `.env` 中设置:

```ini
SAM_MODEL_TYPE=vit_l    # 默认值，4~8GB 显存 (推荐 6GB)
# 或
SAM_MODEL_TYPE=vit_b    # 更轻量，2~4GB 显存
```

或在代码中指定:

```python
sam = SAMWrapper(model_type="vit_l")   # 推荐 6GB 显存
sam = SAMWrapper(model_type="vit_b")   # 显存更小时使用
```

**模型下载到了系统 .cache 目录**

项目已自动将所有模型缓存重定向到 `models/` 目录，包括:

| 环境变量 | 重定向到 | 影响的模型 |
|----------|----------|------------|
| `TORCH_HOME` | `models/torch/` | SAM 权重 |
| `HF_HOME` | `models/huggingface/` | GroundingDINO |
| `TRANSFORMERS_CACHE` | `models/huggingface/transformers/` | transformers 缓存 |
| `HF_HUB_CACHE` | `models/huggingface/hub/` | HuggingFace Hub |
| `CLIP_CACHE` | `models/clip/` | CLIP 权重 |

如果仍有模型下载到 `.cache`，运行 `python config.py` 检查环境变量是否正确设置。

**模型下载超时**

SAM 模型权重 (vit_l 约 1.2GB)，网络差时可能下载失败。可手动下载后放入 `models/` 目录:

| 模型 | 文件 | 大小 |
|------|------|------|
| vit_l (SAM1) | sam_vit_l_0b3195.pth | ~1.2GB |
| vit_b (SAM1) | sam_vit_b_01ec64.pth | ~375MB |
| vit_h (SAM1) | sam_vit_h_4b8939.pth | ~2.5GB |

下载后在 `.env` 中指定路径:

```ini
SAM_CHECKPOINT_PATH=./models/sam_vit_h_4b8939.pth
MODEL_DIR=./models
```

### 无 GPU 环境

如果没有 GPU，项目仍然可以运行，只是速度会显著降低:

```ini
DEVICE=cpu
SAM_MODEL_TYPE=vit_b    # 强烈建议用轻量模型
```

CPU 模式下处理一张 512x512 的影像可能需要数分钟（GPU 下约几秒）。
