### 常见错误与解决方案

#### ModuleNotFoundError: No module named 'samgeo'

**原因**: 未安装 `samgeo` 包。

**解决**:
```bash
pip install samgeo
# 或同时安装
pip install geoai-py
```

#### ModuleNotFoundError: No module named 'geoai'

**原因**: 未安装 `geoai-py` 包（Grounded-SAM 功能需要）。

**解决**:
```bash
pip install geoai-py
```

注意: `geoai-py` 和 `samgeo` 都可以独立使用。如果只需要基础 SAM 功能，`samgeo` 即可。文本提示功能（Grounded-SAM）在两者都不可用时会尝试三级回退。

#### Cannot copy out of meta tensor; no data!

**原因**: 文本提示或整图文本处理会加载 GroundingDINO / SAM 文本模型。`geoai>=0.40` 的 `GroundedSAM` 走 Transformers pipeline 时，部分环境会把模型先放到 meta tensor，再 `.to(cuda)`，从而触发：

```text
Cannot copy out of meta tensor; no data!
Please use torch.nn.Module.to_empty() instead of torch.nn.Module.to()
```

同时，如果 HuggingFace 缓存落到 `C:\Users\...\ .cache\huggingface`，还可能出现权限拒绝或残缺缓存。

**当前项目处理**:

- `GroundedSAMWrapper` 优先使用 `samgeo.text_sam.LangSAM` 和项目内 SAM 权重。
- HuggingFace 缓存统一重定向到 `models/huggingface/`。
- `geoai.segment.GroundedSAM` 仅作为回退路径。

**排查**:

```powershell
python config.py
python -c "from geoai_sam.grounded_sam import GroundedSAMWrapper; g=GroundedSAMWrapper('vit_l'); print(g._resolve_sam_checkpoint())"
```

如果仍失败，优先检查 `models/huggingface/` 是否存在未完成下载，或临时改用框标注 / 自动模式验证基础 SAM 是否正常。

#### CUDA out of memory

**原因**: GPU 显存不足以运行选定的模型。

**解决**:

方案一 — 换用小模型 (推荐 6GB 显存用 vit_l):
```python
# .env 中
SAM_MODEL_TYPE=vit_l   # 4~8GB，默认值
# 或更小
SAM_MODEL_TYPE=vit_b   # 2~4GB
```

方案二 — 使用 CPU:
```python
# .env 中
DEVICE=cpu
```

方案三 — 代码中动态切换:
```python
import torch
device = "cuda" if torch.cuda.is_available() else "cpu"
sam = SAMWrapper(model_type="vit_l", device=device)  # 6GB 显存推荐
```

#### rasterio._err.CPLE_OpenFailedError

**原因**: 影像文件路径不存在或格式不支持。

**排查**:
```python
import os
print(os.path.exists("西安19级.tif"))   # 应为 True

import rasterio
with rasterio.open("西安19级.tif") as src:
    print(f"尺寸: {src.width}x{src.height}")
    print(f"波段: {src.count}")
    print(f"CRS: {src.crs}")
```

#### RuntimeError: CUDA error: device-side assert triggered

**原因**: 通常是坐标越界（点/框坐标超出了影像范围）。

**排查**:
```python
import rasterio
with rasterio.open("西安19级.tif") as src:
    print(f"影像尺寸: {src.width} x {src.height}")

# 确保所有坐标都在范围内
# 例如影像 1024x1024，坐标不应超过 1024
```

#### ImportError: libgdal

**原因**: `rasterio` 依赖 GDAL 底层库，在 Windows 上 pip 安装可能缺失动态库。

**解决**:
```bash
conda install -c conda-forge rasterio geopandas pyproj
```

如果 conda 和 pip 安装冲突，建议全部用 conda:
```bash
conda install -c conda-forge rasterio geopandas shapely pyproj scikit-image scipy
```

#### 模型下载超时/失败

**原因**: SAM 模型权重文件较大 (vit_l 约 1.2GB)，网络差时可能失败。

**解决**: 手动下载后放入 `models/` 目录:

```bash
# SAM1 vit_l (推荐，6GB 显存)
wget https://dl.fbaipublicfiles.com/segment_anything/sam_vit_l_0b3195.pth -P models/

# SAM1 vit_b (更轻量)
wget https://dl.fbaipublicfiles.com/segment_anything/sam_vit_b_01ec64.pth -P models/
```

然后在 `.env` 中指定:
```ini
SAM_CHECKPOINT_PATH=./models/sam_vit_h_4b8939.pth
```

### 性能调优

#### 速度优化

| 优化项 | 做法 | 效果 |
|--------|------|------|
| 换用小模型 | `model_type="vit_b"` | 速度提升 3~5x |
| 降低自动分割密度 | `points_per_side=16` | 速度提升 4x |
| 关闭 multimask_output | `multimask_output=False` | 速度提升约 30% |
| 使用 GPU | 确保 CUDA 可用 | 比 CPU 快 10~50x |
| 裁剪 ROI | 先裁剪感兴趣区域再分割 | 大幅减少计算量 |

#### 显存优化

| 优化项 | 做法 | 效果 |
|--------|------|------|
| 换用小模型 | `vit_b` (2~4GB) | 显存减少 5x |
| 分块处理 | 将大影像分成小块逐块处理 | 显存占用恒定 |
| 及时释放 | `del sam; torch.cuda.empty_cache()` | 释放 GPU 内存 |

### 调试技巧

#### 1. 用 vit_l / vit_b 快速验证

项目默认使用 `vit_l` (1.2GB)，已适配 6GB 显存。如需进一步加速可换 `vit_b`:

```python
# 调试时 (最快)
sam = SAMWrapper(model_type="vit_b")

# 日常使用 (默认)
sam = SAMWrapper(model_type="vit_l")
```

#### 2. 查看中间结果

每个关键步骤后保存中间文件:

```python
sam.save_masks("output/debug/step1_raw_mask.tif")
# 后处理后
processor.save("output/debug/step2_clean_mask.tif", reference_image=image_path)
# 矢量化后
vectorizer.save("output/debug/step3_polygons.geojson")
```

#### 3. 对比可视化

使用 `MaskPostProcessor.visualize()` 生成原始 vs 处理后的对比图:

```python
processor.visualize(
    original_image=img_array,
    save_path="output/debug/comparison.png",
)
```

#### 4. 操作日志

`MaskPostProcessor` 会记录每一步操作:

```python
for op in processor.get_operations_log():
    print(op)
# 输出: {'operation': 'remove_small_objects', 'min_size': 200, 'pixels_removed': 1523}
```

#### 5. Mask 统计

```python
stats = processor.get_statistics()
print(f"目标数: {stats['num_objects']}")
print(f"总面积: {stats['foreground_pixels']}")
print(f"覆盖率: {stats['coverage_ratio']:.4f}")
```

### 调试清单

运行新影像前，建议检查:

1. 影像文件存在且能被 rasterio 打开
2. `python config.py` 输出配置符合预期
3. `quick_start.py` 能正常运行完毕
4. GPU 显存足够（或已设为 CPU 模式）
5. 点/框坐标在影像范围内
6. 后处理参数适合当前场景（参考 04/05 文档的调参表）
