# 点标注参数与结果说明

本文说明 Web 标注平台中“点标注”的完整处理链路，包括前端交互、接口参数、后端坐标转换、SAM 推理结果、Mask 后处理参数和常见错误。

## 1. 使用流程

1. 在左侧“影像文件”中加载 GeoTIFF 影像。
2. 切换到“点标注”模式。
3. 在地图上单击添加前景点。
4. 按住 `Shift` 并单击添加背景点。
5. 双击地图或点击“执行点分割”按钮触发分割。
6. 后端返回 PNG 格式的半透明 Mask，前端叠加到影像范围上显示。

点标注适合目标边界较清晰、只需要少量交互即可确定对象的场景。前景点表示“这里属于目标”，背景点表示“这里不属于目标”。

## 2. 前端提交参数

点标注调用接口：

```http
POST /api/predict/point
```

请求体结构：

```json
{
  "session_id": "当前影像会话 ID",
  "points": [
    [107.123456, 34.123456],
    [107.123789, 34.123789]
  ],
  "labels": [1, 0]
}
```

字段说明：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `session_id` | string | `/api/image/load` 返回的会话 ID，用于找到当前影像和模型实例 |
| `points` | number[][] | 点坐标列表。前端发送 WGS84 经纬度，即 `[lon, lat]` |
| `labels` | number[] | 点标签列表，必须与 `points` 等长 |

`labels` 取值：

| 值 | 含义 | 前端操作 |
| --- | --- | --- |
| `1` | 前景点，目标区域内部 | 单击 |
| `0` | 背景点，非目标区域 | `Shift` + 单击 |

后端会校验：

| 校验项 | 失败响应 |
| --- | --- |
| `points` 不能为空 | HTTP 400，提示至少需要 1 个提示点 |
| `points` 与 `labels` 数量一致 | HTTP 400，提示点坐标数量必须与标签数量一致 |
| `session_id` 有效 | HTTP 404，会话不存在 |

## 3. 坐标转换

前端地图使用 Web Mercator 显示影像，但提交给后端的是 WGS84 经纬度。

后端处理步骤：

1. 读取当前影像的 CRS、仿射变换、宽高和 bounds。
2. 如果影像 CRS 不是 `EPSG:4326`，使用 `rasterio.warp.transform` 将 WGS84 点转换到影像 CRS。
3. 使用影像仿射变换的逆变换，将地理坐标转换为像素坐标 `(col, row)`。
4. 将坐标裁剪到影像有效范围内，避免越界。

转换后的点会传给 SAM：

```python
masks = sam.generate_masks_by_points(
    points=pixel_points,
    point_labels=labels,
)
```

## 4. SAM 推理参数

当前 Web 后端点标注使用 `SAMService.predict_by_points()`，默认参数如下：

| 参数 | 当前值 | 说明 |
| --- | --- | --- |
| `model_type` | `vit_l` | 默认模型类型，可在加载影像时选择 `vit_b`、`vit_l`、`vit_h` |
| `sam_version` | `sam1` | 默认 SAM 版本，可在加载影像时选择 |
| `automatic` | `False` | 点标注使用交互式提示模式，不使用自动全图分割 |
| `multimask_output` | `True` | 底层 SAM 默认输出多个候选 Mask |

模型类型影响：

| 模型 | 特点 |
| --- | --- |
| `vit_b` | 速度快，占用显存低，边界精度相对弱 |
| `vit_l` | 默认推荐，速度和效果较均衡 |
| `vit_h` | 精度潜力更高，但显存和耗时更高 |

## 5. 结果结构

SAM 返回的是 numpy 数组形式的 Mask。常见形状包括：

| 形状 | 含义 |
| --- | --- |
| `(H, W)` | 单个二值 Mask |
| `(N, H, W)` | 多个候选 Mask |
| `(1, N, H, W)` | 部分 SAM 封装返回的批量候选 Mask |

后端会把不同封装返回的结果统一为非空 numpy 数组。如果结果为空或为 `None`，接口返回 HTTP 400，并提示“未生成 Mask”。

前端显示时，后端不会直接返回原始 numpy 数组，而是调用 `_downsample_mask_png()`：

1. 若存在多个候选，默认取第一个 mask 平面。
2. 将 mask 转为 0/255 二值图。
3. 最大边降采样到 2048 像素以内，降低浏览器叠加压力。
4. 转为绿色半透明 RGBA PNG。
5. 前端把 PNG 作为 `ImageStatic` 图层叠加到影像范围上。

## 6. 后处理参数

点标注生成 Mask 后，可以执行“Mask 后处理”。接口：

```http
POST /api/postprocess
```

请求体示例：

```json
{
  "session_id": "当前影像会话 ID",
  "min_size": 200,
  "fill_holes": true,
  "smooth_sigma": 1.5,
  "opening_radius": 2,
  "closing_radius": 3
}
```

参数说明：

| 参数 | 默认值 | 作用 |
| --- | --- | --- |
| `min_size` | `200` | 移除小于该像素面积的孤立斑块 |
| `fill_holes` | `true` | 填充目标内部孔洞 |
| `smooth_sigma` | `1.5` | 高斯平滑强度，降低边缘毛刺 |
| `opening_radius` | `2` | 开运算半径，去除细小噪声和毛刺 |
| `closing_radius` | `3` | 闭运算半径，连接窄缝和小断裂 |

处理顺序由 `MaskPostProcessor.default_pipeline()` 控制，当前顺序为：

```text
load -> remove_small_objects -> fill_holes -> opening -> closing -> smooth
```

## 7. 常见问题

| 现象 | 可能原因 | 处理方式 |
| --- | --- | --- |
| 点标注返回 400 | 没有点或点/标签数量不一致 | 至少添加一个前景点，确认标签数量与点数量一致 |
| 点标注返回 503 | 模型依赖不可用 | 检查 `torch`、`samgeo`、模型 checkpoint 是否安装完整 |
| 分割结果偏移 | 影像 CRS 或前端显示范围转换异常 | 检查影像 CRS、bounds 和前端叠加范围 |
| Mask 很粗糙 | 点太少或提示不明确 | 增加前景点，使用背景点排除干扰区域 |
| 结果包含多个目标 | 前景点过少或背景约束不足 | 添加背景点，或改用框标注限定范围 |
| 后处理后目标变小 | `min_size` 或 `opening_radius` 过大 | 降低对应参数 |

## 8. 与框标注、文本标注的区别

| 模式 | 输入 | 适合场景 |
| --- | --- | --- |
| 点标注 | 前景点、背景点 | 快速交互式分割单个对象 |
| 框标注 | 一个矩形范围 | 目标范围明确、需要限制 SAM 搜索区域 |
| 文本标注 | 文本提示词 | 希望由 GroundingDINO/LangSAM 自动检测目标 |

点标注最依赖人工提示质量。前景点应落在目标内部稳定区域，背景点应落在容易混淆的邻近区域，而不是远离目标的位置。
