## 01 - 项目介绍

### 背景

遥感影像标注是 GeoAI（地理空间人工智能）的核心前置环节。无论是建筑物提取、水体识别还是道路检测，深度学习模型都需要大量高质量的 Polygon 标签作为训练数据。传统流程完全依赖人工在 GIS 软件中逐像素勾画，一个熟练标注员处理一平方公里的高分辨率影像可能需要数小时。

SAM（Segment Anything Model）的出现改变了这一局面。Meta 于 2023 年发布的 SAM 是一个通用的图像分割基础模型，它接受点或框作为"提示"（Prompt），输出对应的分割 Mask。将 SAM 应用于遥感影像，再配合 GroundingDINO 等目标检测模型，可以实现"文本描述 → 自动检测 → 精确分割"的全自动工作流。

### 本项目做什么

本项目是一个**完整的遥感半自动标注工程**，不是单个脚本，而是一个可复用的工具包。它封装了从影像加载到标签导出的全部环节:

1. **模型调用层**: 统一封装 SAM、SAM2、SAM3 三种版本的模型接口，通过 `SAMWrapper` 类屏蔽版本差异
2. **智能检测层**: 集成 GroundingDINO + CLIP，通过 `GroundedSAMWrapper` 实现文本驱动的自动分割
3. **后处理层**: 提供 `MaskPostProcessor` 链式 API，一步完成去噪、填充、平滑、过滤
4. **矢量化层**: 通过 `MaskVectorizer` 将栅格 Mask 转为 GeoJSON / Shapefile / GeoPackage
5. **质量评估层**: `QualityMetrics` 提供 IoU、Dice、Hausdorff 距离等量化指标

### 为什么用 SAM 做遥感标注

对比传统流程和 SAM 辅助流程:

**传统流程**: 影像 → 人工勾画 Polygon → 检查 → 导出标签。人工成本高，一致性差，大面积区域效率极低。

**SAM 辅助流程**: 影像 → SAM 自动分割 → 人工修正 → 生成标签。通常可以减少 70%~90% 的人工标注时间。

SAM 的优势在于: 它是一个预训练的通用分割模型，不需要针对遥感场景额外训练；只需极少的交互（一个点或一个框）就能输出精确的分割边界；配合 GroundingDINO，甚至只需要输入文本描述就能全自动完成。

### 项目目标

- 提供一个**开箱即用**的标注工具包，在 `geoai` conda 环境中安装依赖后即可运行
- 覆盖遥感标注的**完整流程**，从加载影像到导出可用的训练标签
- 支持多种标注模式，让用户根据场景灵活选择点提示、框提示或文本提示
- 代码结构清晰，可作为**二次开发的基础框架**

### 技术栈

项目的底层依赖关系:

```
PyTorch (计算引擎)
    ├── SAM / SAM2 / SAM3 (分割模型)
    ├── GroundingDINO (目标检测)
    └── CLIP (语义匹配)
        ↓
samgeo / geoai-py (遥感接口层)
        ↓
geoai_sam (本项目封装层)
        ↓
demo_*.py / interactive_annotate.py (用户入口)
```

遥感数据处理依赖 `rasterio`（读写 GeoTIFF）、`geopandas`（矢量操作）、`shapely`（几何计算）等地理空间库。
