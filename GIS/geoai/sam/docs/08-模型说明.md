# GeoAI SAM 项目 — 模型说明文档

## 概述

本项目基于 Meta AI 的 **Segment Anything Model (SAM)** 系列模型，支持 SAM1、SAM2 和 SAM3 三个版本，用于遥感影像的半自动标注。

所有模型权重统一存放在项目 `models/` 目录下，不会下载到系统缓存目录。

---

## 模型版本对比

| 特性 | SAM1 | SAM2 (sam2.1) | SAM3 |
|------|------|---------------|------|
| 发布方 | Meta AI (2023.04) | Meta AI (2024.09) | Meta AI (sam2.1 复用) |
| 架构 | ViT (Vision Transformer) | Hiera (分层 Transformer) | 与 SAM2 共享架构 |
| 视频支持 | 不支持 | 支持 (时序分割) | 支持 |
| 精度 | 基准 | 提升 | 同 SAM2 |
| 速度 | 基准 | 更快 | 同 SAM2 |
| 权重文件 | 独立 | 独立 | 复用 SAM2 权重 |

> **说明**: SAM3 在本项目中复用 SAM2.1 系列权重文件。代码中 `sam_version="sam3"` 会加载与 SAM2 相同的检查点，但在推理接口上可能有不同的参数传递方式。

## 模型类型与显存需求

| 模型类型 | 参数量 | 显存需求 | 适用场景 | 推荐度 |
|----------|--------|----------|----------|--------|
| `vit_b` | 91M | 2~4GB | 快速测试、CPU 推理 | 入门 |
| `vit_l` | 308M | 4~8GB | 生产标注、精度与速度平衡 | **推荐** (6GB 显存) |
| `vit_h` | 636M | 8GB+ | 精细标注、建筑物提取 | 高精度 |

## 模型选择建议

### 按显存选择

| GPU 显存 | SAM1 推荐 | SAM2/SAM3 推荐 |
|----------|-----------|----------------|
| 2GB | vit_b (CPU 也可) | vit_b |
| 4GB | vit_b / vit_l | vit_b / vit_l |
| 6GB | **vit_l** (最佳平衡) | **vit_l** |
| 8GB+ | vit_l / vit_h | vit_l / vit_h |
| 12GB+ | vit_h (最高精度) | vit_h |

### 按任务选择

| 任务场景 | 推荐模型 | 原因 |
|----------|----------|------|
| 快速原型验证 | SAM1 vit_b | 下载快，CPU 可运行 |
| 建筑物提取 | SAM1 vit_l | 精度足够，速度快 |
| 精细地物标注 | SAM1 vit_h | 边界精度最高 |
| 视频时序分割 | SAM2 vit_l | 支持视频帧间追踪 |
| 大批量自动标注 | SAM2 vit_l | Hiera 架构更高效 |
| 文本提示标注 | SAM1 vit_l + GroundingDINO | 配合 GroundingDINO 使用 |

## 下载方式

### 方式一：一键下载脚本（推荐）

```bash
# 下载全部模型
python download_models.py

# 仅下载 SAM1
python download_models.py --sam1

# 仅下载 SAM2/SAM3
python download_models.py --sam2

# 仅下载指定级别
python download_models.py --vit_b
python download_models.py --vit_l
python download_models.py --vit_h

# 查看模型列表和下载状态
python download_models.py --list
```

### 方式二：运行时自动下载

在代码中使用 `SAMWrapper` 时，如果本地没有模型文件，会自动下载：

```python
from geoai_sam import SAMWrapper

# 自动下载 vit_l 模型到 models/ 目录
sam = SAMWrapper(model_type="vit_l", sam_version="sam1")
sam.set_image("image.tif")
```

### 方式三：手动下载

如果网络环境不佳，可使用浏览器或下载工具手动下载：

1. 从上方表格中的下载地址下载 `.pth` / `.pt` 文件
2. 将文件放入项目 `models/` 目录

```powershell
# PowerShell 下载示例
Invoke-WebRequest -Uri "https://dl.fbaipublicfiles.com/segment_anything/sam_vit_b_01ec64.pth" -OutFile "models\sam_vit_b_01ec64.pth"
```

```bash
# curl 下载示例
curl -L -o models/sam_vit_b_01ec64.pth https://dl.fbaipublicfiles.com/segment_anything/sam_vit_b_01ec64.pth
```

## 配置说明

通过 `.env` 文件或环境变量配置模型：

```bash
# .env 文件
SAM_VERSION=sam1              # sam1 / sam2 / sam3
SAM_MODEL_TYPE=vit_l          # vit_b / vit_l / vit_h
MODEL_DIR=./models            # 模型存放目录
```

在代码中指定模型：

```python
from geoai_sam import SAMWrapper

# SAM1 + vit_l
sam = SAMWrapper(model_type="vit_l", sam_version="sam1")

# SAM2 + vit_b
sam = SAMWrapper(model_type="vit_b", sam_version="sam2")

# SAM3 + vit_h
sam = SAMWrapper(model_type="vit_h", sam_version="sam3")
```

## 常见问题

### Q: 下载速度慢怎么办？

A: 可尝试以下方法：
1. 使用 VPN 或代理
2. 手动下载文件放入 `models/` 目录
3. 先下载 `vit_b`（最小），跑通流程后再下载更大的模型

### Q: 显存不足怎么办？

A: 
1. 使用更小的模型 (`vit_b`)
2. 设置 `DEVICE=cpu` 使用 CPU 推理（速度较慢）
3. 裁取影像子区域进行处理，避免全图加载

### Q: SAM1 和 SAM2 如何选择？

A: 
- **SAM1**: 生态成熟，兼容性最好，适合大多数遥感标注场景
- **SAM2**: 架构更高效，模型更小但精度相当，适合需要视频时序分割的场景
- 如果不确定，优先使用 SAM1 vit_l

### Q: SAM3 和 SAM2 有什么区别？

A: 在本项目中，SAM3 复用 SAM2.1 系列权重文件。代码层面通过 `sam_version` 参数区分，但底层加载的检查点文件相同。SAM3 的差异主要体现在推理接口的参数传递方式上。
