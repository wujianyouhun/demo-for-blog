## 03 - 项目结构详解

### 目录总览

```
sam/
├── geoai_sam/                  ← 核心包 (5 个模块)
├── demo_*.py                   ← 4 个演示脚本
├── interactive_annotate.py     ← CLI 交互工具
├── quick_start.py              ← 快速入门
├── config.py                   ← 全局配置
├── requirements.txt            ← 依赖清单
├── .env.example                ← 环境变量模板
├── docs/                       ← 文档目录
├── models/                     ← 模型权重 (自动创建)
├── output/                     ← 输出结果 (自动创建)
└── 西安19级.tif                 ← 测试影像
```

### 核心包 geoai_sam/

这是项目的核心，包含 5 个模块，每个模块封装了一个独立的职责。

**`__init__.py` — 包入口**

统一导出 5 个核心类，用户只需 `from geoai_sam import XXX` 即可使用全部功能:

```python
from geoai_sam import (
    SAMWrapper,           # SAM 模型封装
    GroundedSAMWrapper,   # Grounded-SAM 封装
    MaskPostProcessor,    # Mask 后处理
    MaskVectorizer,       # Mask 矢量化
    QualityMetrics,       # 质量评估
)
```

**`core.py` — SAMWrapper**

SAM 模型的统一封装层。核心设计决策是**延迟初始化**: 模型不会在构造函数中立即加载，而是在第一次调用 `set_image()` 时才真正初始化。这有两个好处: 避免 import 时就占用 GPU 显存；允许用户先创建对象、后加载模型。

`SAMWrapper` 同时支持 SAM1、SAM2、SAM3 三个版本，通过 `sam_version` 参数切换。内部自动选择对应的 `samgeo` 接口类 (`SamGeo` / `SamGeo2`)。如果指定的版本不可用，会自动回退到可用的版本。

关键方法:

| 方法 | 用途 |
|------|------|
| `set_image(path)` | 加载遥感影像 |
| `generate_masks_by_points(points, point_labels)` | 点提示分割 |
| `generate_masks_by_box(box)` | 单框提示分割 |
| `generate_masks_by_boxes(boxes)` | 多框批量分割 |
| `generate_masks_auto(...)` | 全图自动分割 |
| `save_masks(path)` | 保存 Mask 到文件 |
| `show_masks(...)` | 可视化 Mask |

**`grounded_sam.py` — GroundedSAMWrapper**

文本驱动的自动分割，是本项目最有生产价值的模块。它整合了 GroundingDINO（文本→检测框）和 SAM（检测框→精确 Mask），实现了零样本目标分割。

内部采用三级回退策略:
1. 优先使用 `geoai.segment.GroundedSAM`（最完整的接口）
2. 回退到 `samgeo.text_sam.LangSAM`（CLIP + SAM）
3. 最终回退到 `samgeo.SamGeo`（基础 SAM）

关键方法:

| 方法 | 用途 |
|------|------|
| `set_image(path)` | 加载影像 |
| `segment_by_text(prompt, ...)` | 文本提示分割 |
| `segment_by_text_with_clip(prompt, ...)` | CLIP 增强分割 |
| `save_results(mask_path, box_path)` | 保存 Mask + 检测框 |

**`postprocess.py` — MaskPostProcessor**

SAM 输出的原始 Mask 通常有毛刺、碎片、孔洞和边界抖动，需要后处理才能作为训练标签。该模块提供链式调用 API:

```python
clean = (MaskPostProcessor()
    .load(raw_mask)
    .remove_small_objects(min_size=200)
    .fill_holes()
    .opening(radius=2)
    .closing(radius=3)
    .smooth(sigma=1.5)
    .get_result())
```

所有操作方法都返回 `self`，可以任意组合。另外提供 `default_pipeline()` 静态方法一键执行推荐的后处理流程。

支持的形态学操作: 去小斑块、孔洞填充、开运算、闭运算、膨胀、腐蚀、高斯平滑、面积过滤、保留最大区域、凸包处理。

**`vectorize.py` — MaskVectorizer**

将二值栅格 Mask 转为矢量 Polygon。核心依赖 `rasterio.features.shapes()` 做栅格转矢量，用 `shapely` 做几何操作，用 `geopandas` 管理属性表和输出。

如果 rasterio/shapely 不可用，会回退到基于 `scipy.ndimage.label()` 的简化矢量化（只生成边界矩形）。

支持的输出格式: GeoJSON (.geojson)、Shapefile (.shp)、GeoPackage (.gpkg)、GeoParquet (.parquet)。

**`quality.py` — QualityMetrics**

标签质量的量化评估。所有方法均为静态方法，可直接调用:

| 指标 | 公式 | 说明 |
|------|------|------|
| IoU | |A∩B| / |A∪B| | 最核心的分割精度指标 |
| Dice | 2|A∩B| / (|A|+|B|) | 对小目标更友好 |
| Precision | TP / (TP+FP) | 预测为正的准确率 |
| Recall | TP / (TP+FN) | 真值的召回率 |
| F1 | 2·P·R / (P+R) | P 和 R 的调和平均 |
| 面积误差 | |area_pred - area_gt| / area_gt | 面积偏差 |
| Hausdorff 距离 | max(d(A→B), d(B→A)) | 边界精度 |

`evaluate()` 方法一次性计算所有指标，`print_report()` 输出格式化的评估报告。

### 演示脚本

每个 `demo_*.py` 都是独立可运行的完整流程演示:

| 脚本 | 演示内容 | 输出目录 |
|------|----------|----------|
| `demo_point_prompt.py` | 单点/多点/正负联合标注 + 后处理 + 矢量化 | output/point_prompt/ |
| `demo_box_prompt.py` | 单框/多框/框+点联合标注 | output/box_prompt/ |
| `demo_text_prompt.py` | Grounded-SAM + CLIP + 多类别提取 | output/text_prompt/ |
| `demo_postprocess.py` | 完整后处理流程 + 可视化对比 | output/postprocess/ |

### 工具脚本

**`interactive_annotate.py`** — 完整的 CLI 交互工具，支持 `argparse` 命令行参数，也支持交互式的坐标输入。它是面向最终用户的主要入口。

**`quick_start.py`** — 最短的可运行示例，约 50 行代码完成从加载影像到导出 Polygon 的完整流程。适合新手快速验证环境是否正常。

**`config.py`** — 全局配置中心。优先读取 `.env` 文件，其次使用环境变量，最后使用代码中的默认值。直接运行 `python config.py` 可查看当前所有配置。

### 数据文件

**`西安19级.tif`** — 测试用遥感影像，GeoTIFF 格式，用于所有演示脚本和快速入门。

**`models/`** — 模型权重存放目录。首次运行时 SAM 权重会自动下载到此目录，后续运行直接从本地加载。

**`output/`** — 所有运行结果的输出根目录。每个演示脚本在其中创建独立的子目录。
