# 数据准备指南

弱标签（weak-label）和矢量标签（vector-label）模式都需要准备双时相影像数据，矢量标签模式还需要额外准备变化矢量标注。模拟样本（synthetic）模式无需任何真实数据，适合快速跑通流程。本文档详细介绍每种数据的准备方法。

---

## 一、双时相影像准备

弱标签和矢量标签模式都需要两幅**同一区域、不同时期**的遥感影像，且必须空间对齐。

### 1.1 数据来源

#### 方式一：通过项目内置脚本下载 Sentinel-2 影像（推荐）

项目集成了 Planetary Computer STAC API，可自动下载 Sentinel-2 L2A 影像并完成空间对齐。

**命令行方式：**

```bash
python scripts/download_data.py \
  --region beijing \
  --date-a 2022-06-01 \
  --date-b 2023-06-01 \
  --cloud 20
```

**Web 界面方式：**

启动项目后，在前端「数据下载」面板选择区域、日期和云量，点击下载。

**可选预设区域：**

| 区域标识 | 名称 | 范围（经纬度） | EPSG |
|---------|------|---------------|------|
| `beijing` | 北京 | 116.20°E~116.60°E, 39.75°N~40.05°N | 32650 |
| `shanghai` | 上海浦东 | 121.40°E~121.70°E, 31.10°N~31.35°N | 32651 |
| `shenzhen` | 深圳 | 113.85°E~114.15°E, 22.45°N~22.65°N | 32650 |
| `dubai` | 迪拜 | 55.10°E~55.40°E, 25.05°N~25.25°N | 32640 |
| `san_francisco` | 旧金山 | -122.52°E~-122.35°E, 37.70°N~37.83°N | 32610 |

也可使用 `--bbox` 指定自定义范围：

```bash
python scripts/download_data.py \
  --bbox 116.30 39.85 116.45 39.95 \
  --date-a 2022-06-01 \
  --date-b 2023-06-01
```

**下载参数说明：**

| 参数 | 默认值 | 说明 |
|------|-------|------|
| `--region` | beijing | 预设区域名称 |
| `--bbox` | 无 | 自定义范围 [left, bottom, right, top]（WGS84 经纬度） |
| `--date-a` | 2022-06-01 | 时相 A 日期（YYYY-MM-DD），实际搜索该月整月 |
| `--date-b` | 2023-06-01 | 时相 B 日期 |
| `--cloud` | 20.0 | 最大云量百分比，越低影像越清晰但可选范围越小 |
| `--name` | 自动生成 | 输出文件名前缀 |

**下载产出：**

```
data/raw/
├── time_a/
│   └── s2_116.20_39.75_A.tif          # 时相 A 影像（RGB 三通道，10m 分辨率）
└── time_b/
    ├── s2_116.20_39.75_B.tif           # 时相 B 原始影像
    └── s2_116.20_39.75_B_aligned.tif   # 时相 B 对齐影像（使用此文件）
```

> **注意**：时相 B 会生成两个文件，带 `_aligned` 后缀的是与 A 空间对齐后的版本，制作样本时**必须使用对齐版本**。

#### 方式二：使用自有影像数据

如果你已有遥感影像，可以直接放入数据目录：

```
data/raw/time_a/your_image_a.tif
data/raw/time_b/your_image_b_aligned.tif
```

**自有影像的要求：**

| 要求 | 说明 |
|------|------|
| 格式 | GeoTIFF（.tif），必须包含地理参考信息（CRS + Transform） |
| 通道数 | 至少 3 通道（RGB），不足 3 通道时程序自动复制第一通道补齐 |
| 数据类型 | uint8 最佳；其他类型会自动做 2%~98% 百分位拉伸转 uint8 |
| 空间对齐 | 两幅影像必须 CRS 一致、分辨率一致、像素对齐（见下方说明） |
| 范围一致 | 两幅影像覆盖的地理范围应相同或高度重叠 |

### 1.2 空间对齐

双时相影像必须**像素级对齐**，即同一像素坐标 (row, col) 在两幅影像中对应同一地面位置。如果自有影像未对齐，可使用以下方法：

**方法一：使用项目内置对齐功能**

将时相 A 作为参考，对时相 B 进行重投影对齐：

```python
import rasterio
from rasterio.warp import reproject, Resampling

ref_path = "data/raw/time_a/image_A.tif"
src_path = "data/raw/time_b/image_B.tif"
out_path = "data/raw/time_b/image_B_aligned.tif"

with rasterio.open(ref_path) as ref:
    rt, rc, rw, rh = ref.transform, ref.crs, ref.width, ref.height

with rasterio.open(src_path) as src:
    aligned = np.zeros((src.count, rh, rw), dtype=src.dtypes[0])
    for i in range(src.count):
        reproject(
            source=rasterio.band(src, i + 1),
            destination=aligned[i],
            src_transform=src.transform, src_crs=src.crs,
            dst_transform=rt, dst_crs=rc,
            resampling=Resampling.bilinear,
        )

profile = {
    "driver": "GTiff", "dtype": "uint8",
    "width": rw, "height": rh,
    "count": aligned.shape[0],
    "crs": rc, "transform": rt, "compress": "lzw",
}
with rasterio.open(out_path, "w", **profile) as dst:
    dst.write(aligned)
```

**方法二：使用 GDAL 命令行**

```bash
gdalwarp -t_srs EPSG:32650 -tr 10 10 -te 449760 4414080 463200 4427520 \
  image_B.tif image_B_aligned.tif
```

参数说明：
- `-t_srs`：目标坐标系
- `-tr`：目标分辨率（米）
- `-te`：目标范围（与 A 影像一致）

### 1.3 影像质量检查

制作样本前建议检查影像质量：

```python
import rasterio
import numpy as np

def check_image(path):
    with rasterio.open(path) as src:
        print(f"尺寸: {src.width} x {src.height}")
        print(f"波段: {src.count}")
        print(f"CRS: {src.crs}")
        print(f"分辨率: {src.res}")
        print(f"范围: {src.bounds}")

        data = src.read()
        valid = data[data > 0]
        if len(valid) > 0:
            print(f"有效像素: {len(valid)} / {data.size}")
            print(f"值域: [{valid.min()}, {valid.max()}]")
            nodata_ratio = 1 - len(valid) / data.size
            print(f"无数据比例: {nodata_ratio:.1%}")
        else:
            print("警告: 影像全部为 0，可能下载失败")

check_image("data/raw/time_a/s2_beijing_A.tif")
check_image("data/raw/time_b/s2_beijing_B_aligned.tif")
```

**常见问题：**

| 问题 | 原因 | 解决方法 |
|------|------|---------|
| 影像全部为 0 | 下载失败或 bbox 超出范围 | 检查网络，确认 bbox 在有效范围内 |
| 无数据比例高 | 云遮挡或 bbox 在影像边缘 | 降低云量阈值，调整 bbox |
| 两幅影像尺寸不同 | 未空间对齐 | 使用对齐后的 `_aligned.tif` 文件 |
| CRS 不一致 | 不同来源影像 | 用 gdalwarp 统一到同一 CRS |

## 二、模拟样本准备（synthetic 模式）

模拟样本模式**不需要任何真实影像或标注数据**，通过程序自动生成随机的双时相影像和变化标签，适合快速跑通训练流程、验证代码逻辑和教学演示。

> **注意**：模拟样本不包含真实地物信息，训练出的模型无法用于实际变化检测。正式训练请使用弱标签或矢量标签模式。

### 2.1 生成方法

```bash
python scripts/generate_sample.py --mode synthetic --num-samples 100
```

**参数说明：**

| 参数 | 默认值 | 说明 |
|------|-------|------|
| `--mode` | synthetic | 模式选择，synthetic 为模拟样本 |
| `--num-samples` | 100 | 生成样本数量 |
| `--tile-size` | 256 | 每张样本的像素尺寸 |
| `--keep-existing` | False | 追加样本而非清空已有样本 |

### 2.2 生成原理

模拟样本的生成过程（代码位于 [cdd/sample_builder.py](file:///f:/study/demo-for-blog/GIS/geoai/change-detection/cdd/sample_builder.py#L33-L72) 的 `generate_synthetic_samples` 函数）：

**步骤 1：生成时相 A 基底影像**

- 生成随机像素值底图：`rng.randint(50, 200, (3, 256, 256))`
- 在底图上随机放置 3~7 个矩形色块，模拟建筑物等地物：
  - 位置：随机 (cx, cy)
  - 尺寸：随机宽高 (10~40 像素)
  - 颜色：随机 RGB (100~255)

**步骤 2：生成时相 B 影像和变化标签**

- 复制时相 A 的底图作为时相 B 的基础
- 在时相 B 上随机放置 1~5 个矩形色块，模拟变化区域：
  - 位置：随机 (cx, cy)
  - 尺寸：随机宽高 (8~30 像素)
  - 颜色：随机 RGB (50~255)
  - 同时在标签数组对应位置标记为 1（变化）
  - 50% 概率同步修改时相 A 的对应区域（模拟双向变化）

**步骤 3：添加高斯噪声**

- 对时相 A 和时相 B 各添加高斯噪声：`N(0, 5)`
- 模拟遥感影像中的传感器噪声，使样本更接近真实数据

**步骤 4：写入 GeoTIFF 三元组**

每个样本生成三个文件：
- `data/samples/time_a/sample_XXXX.tif` — 时相 A 影像
- `data/samples/time_b/sample_XXXX.tif` — 时相 B 影像
- `data/samples/labels/sample_XXXX.tif` — 变化标签（0=未变化，1=变化）

所有文件均包含地理参考信息（CRS: EPSG:4326，范围: 116.3°~116.4°E, 39.8°~39.9°N），确保下游训练代码可正常读取。

### 2.3 产出结构

```
data/samples/
├── time_a/
│   ├── sample_0000.tif
│   ├── sample_0001.tif
│   └── ...（共 num_samples 张）
├── time_b/
│   ├── sample_0000.tif
│   ├── sample_0001.tif
│   └── ...
└── labels/
    ├── sample_0000.tif
    ├── sample_0001.tif
    └── ...
```

### 2.4 与其他模式的对比

| 特性 | synthetic（模拟） | weak-label（弱标签） | vector-label（矢量标签） |
|------|------------------|--------------------|-----------------------|
| 是否需要真实影像 | 不需要 | 需要 | 需要 |
| 是否需要标注数据 | 不需要 | 不需要（自动生成） | 需要（人工标注） |
| 标签精度 | 100%（人工构造） | 中等（模型预测） | 高（人工标注） |
| 样本真实性 | 低（随机色块） | 高（真实地物） | 高（真实地物） |
| 适用场景 | 教学演示、代码验证 | 快速生成大量样本 | 正式监督训练 |

## 三、矢量标签准备（vector-label 模式专用）

矢量标签模式需要额外提供人工标注的变化区域矢量文件。

### 3.1 矢量标签格式

| 项目 | 要求 |
|------|------|
| 文件格式 | GeoJSON（推荐）、GeoPackage（.gpkg）、Shapefile（.shp） |
| 几何类型 | Polygon 或 MultiPolygon |
| 坐标系 | 任意 CRS，程序自动重投影到影像坐标系 |
| 属性字段 | 无特殊要求，仅使用几何信息 |

**推荐使用 GeoJSON**，因为它是纯文本格式，便于版本管理和查看编辑。

### 3.2 标注内容

每个多边形表示一处变化区域，包括但不限于：

- 新建建筑
- 拆除建筑
- 道路新建/扩建
- 地表覆盖类型变化（如农田→建筑用地）
- 其他肉眼可辨的变化

### 3.3 标注工具

#### QGIS（推荐）

1. 打开 QGIS，加载双时相影像
2. 菜单 → 图层 → 新建 → 新建临时图层
3. 几何类型选择「多边形」
4. 在卷帘对比模式下逐个勾绘变化区域
5. 导出为 GeoJSON：右键图层 → 导出 → 另存为 → 格式选 GeoJSON

#### geojson.io（在线轻量工具）

1. 访问 https://geojson.io
2. 使用多边形工具在地图上勾绘变化区域
3. 导出 GeoJSON

> **注意**：geojson.io 使用 WGS84 (EPSG:4326) 坐标系，与大多数遥感影像 CRS 不同，但程序会自动处理重投影。

#### 其他工具

- **JOSM**：OpenStreetMap 编辑器，支持 GeoJSON 导出
- **FME**：商业 ETL 工具，适合批量处理
- **Python + geopandas**：编程方式创建矢量标注

### 3.4 标注规范

#### 基本原则

1. **只标注确定的变化**：不确定是否变化的区域不要标注，宁可漏标不可误标
2. **边界贴合实际**：多边形边界应贴合变化区域的实际轮廓，避免过大或过小
3. **避免重叠**：同一变化区域只画一个多边形，不要重复标注
4. **覆盖完整**：尽量标注影像中所有可见的变化区域，保证正负样本平衡

#### 标注精度建议

| 变化类型 | 标注建议 |
|---------|---------|
| 大型建筑群 | 按单体建筑分别标注，或按建筑群整体标注 |
| 道路 | 沿道路轮廓勾绘，包含路面及路肩 |
| 小型建筑 | 确保多边形面积 ≥ 3×3 像素（Sentinel-2 10m 分辨率下约 30m×30m） |
| 拆除区域 | 标注拆除前的建筑轮廓，而非裸地轮廓 |

#### 示例 GeoJSON

```json
{
  "type": "FeatureCollection",
  "name": "change_labels",
  "crs": {
    "type": "name",
    "properties": { "name": "urn:ogc:def:crs:EPSG::4326" }
  },
  "features": [
    {
      "type": "Feature",
      "properties": { "change_type": "new_building" },
      "geometry": {
        "type": "Polygon",
        "coordinates": [[
          [116.350, 39.850],
          [116.355, 39.850],
          [116.355, 39.855],
          [116.350, 39.855],
          [116.350, 39.850]
        ]]
      }
    },
    {
      "type": "Feature",
      "properties": { "change_type": "demolished" },
      "geometry": {
        "type": "Polygon",
        "coordinates": [[
          [116.360, 39.860],
          [116.363, 39.860],
          [116.363, 39.863],
          [116.360, 39.863],
          [116.360, 39.860]
        ]]
      }
    }
  ]
}
```

### 3.5 标注质量检查

制作样本前，验证矢量标签与影像的配准关系：

```python
import geopandas as gpd
import rasterio
from rasterio.plot import show
import matplotlib.pyplot as plt

image_path = "data/raw/time_a/s2_beijing_A.tif"
vector_path = "labels/change.geojson"

gdf = gpd.read_file(vector_path)
print(f"标注数量: {len(gdf)}")
print(f"标注 CRS: {gdf.crs}")
print(f"标注范围: {gdf.total_bounds}")

with rasterio.open(image_path) as src:
    print(f"影像 CRS: {src.crs}")
    fig, ax = plt.subplots(figsize=(10, 10))
    show(src, ax=ax)
    gdf.to_crs(src.crs).plot(ax=ax, facecolor="red", edgecolor="yellow", alpha=0.4)
    plt.title("矢量标签叠加检查")
    plt.savefig("label_check.png", dpi=150)
```

**检查要点：**

- 矢量多边形是否与影像中的变化区域位置吻合
- 是否有遗漏的变化区域
- 是否有误标注（未变化区域被标注为变化）
- CRS 是否正确（如果多边形位置偏移很大，可能是坐标系问题）

## 五、常见问题

### Q: 下载影像时提示"未找到影像"

Sentinel-2 重访周期为 5 天，但受云量限制，某些月份可能没有低云量影像。建议：
- 放宽云量阈值（如 `--cloud 30`）
- 扩大日期搜索范围（选择不同月份）
- 检查 bbox 是否在 Sentinel-2 覆盖范围内（全球陆地基本覆盖）

### Q: 弱标签样本数量为 0

说明 ChangeStar 在当前影像上未检测到足够变化，可能原因：
- 两幅影像时间间隔太短，变化不明显
- 阈值 `--threshold` 设置过高，降低阈值（如 0.3）可增加检出率
- 影像质量差（云遮挡、传感器异常）

### Q: 矢量标签样本数量为 0

说明栅格化后的标签中变化像素少于 `min_change_pixels`（默认 20），可能原因：
- 矢量标注区域太小（Sentinel-2 10m 分辨率下 20 像素 ≈ 2000m²）
- 矢量标注与影像空间不匹配（坐标系错误或范围不一致）
- 降低 `--min-change-pixels` 可保留更小的变化区域

### Q: 自有影像如何确认是否对齐

```python
import rasterio

with rasterio.open("image_A.tif") as a, rasterio.open("image_B.tif") as b:
```
